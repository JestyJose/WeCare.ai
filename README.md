# 💙 WeCare: An AI that Cares...

**WeCare** is a next-generation, proactive healthcare command center initially designed for Hackcraft 2026. Evolving from a simple dashboard, it is now a fully state-managed, deeply empathetic life-tracking system. WeCare doesn't just display numbers; it mathematically tracks risk, seamlessly alerts you to immediate dangers, natively follows up on worsening symptoms, and intelligently provides local, human-centered help.

---

## 🌟 Core Features

### 1. 🧠 Smart Risk Engine & Progression Tracking
The heart of WeCare is an autonomous 0–100 Risk Engine.
- **Vital Parsing:** The system constantly combs `Heart Rate`, `Blood Pressure`, `SpO2`, and `Temperature`. Normal vitals are displayed beautifully, but breaching *Warning Thresholds* mathematically increments your risk score silently. 
- **Intelligent Symptom Follow-Up (USP):** When you report feeling "Not well", WeCare remembers the symptoms. One week later, instead of a blank check-in, the system asks you exactly how your exact stated symptoms have progressed (Improved, Same, or Worse). It tracks your multi-week trajectory and raises flags if your healing trend falls flat.

### 2. 🚨 Global Alert Notification Queue
- A global `AlertManager` intelligently intercepts mathematical thresholds.
- **Non-Blocking Toasts:** Gracefully fades in from the bottom right for trivial activities like data saves without interrupting user flow.
- **Warning Notifications:** Small, dismissible pop-ups on the top right that notify the user to click into their Risk page if their health drifts out of balance.
- **Critical Countdowns:** If severe life conditions (e.g. `Score >= 100`, `Fall Detection`, `Critical SpO2 Drop`) are hit, an un-dismissible, screen-shaking countdown dominates the UI. When 5 seconds run out, the ambulance fires.

### 3. 🤝 Empathy & Local Support (ASHA Integration)
WeCare prioritizes keeping severe logic isolated from minor sickness. 
- **AI Explainability:** When users input mildly annoying symptoms, WeCare provides dynamic, animated bars beautifully contextualizing exactly *why* it tracked certain points.
- **Local ASHA Worker Binding:** If symptoms are "Moderate", instead of dropping alarms onto the user, WeCare gently surfaces an integrated contact card linking simulated community ASHA (Accredited Social Health Activist) providers for real, local human support.

### 4. 🚑 Emergency Ambulance & Recovery Flow
- Once the Risk Score breaches 100+, the UI locks. A siren-themed Emergency Modal immediately triggers. 
- **Post-Emergency Recovery**: The AI recognizes emergencies have been handled. It instantly ceases continuous alarms, places a red banner over the system, and forces the user into a "Medical Cooldown" until they click `Reset Risk` to securely guarantee medical stability.

### 5. 🤖 Google Gemini Medical Triage Assessor
An isolated endpoint (`SymptomChecker`) that packages complex patient inputs and routes them securely mapped through a clinical Prompt Engine to the `gemini-2.5-flash` model structure, returning safely filtered and professional assessments.

## 🔁 System Architecture & Workflow

WeCare operates entirely natively using an advanced front-end state loop backed by browser `localStorage` for zero-latency hackathon performance.

1. **Initialization & Hydration**: Users authenticate, and `Dashboard.jsx` pulls all historical arrays (Vitals, Wellness, Symptoms, Alerts, Emergency logs, Risk Score) from memory. If data is empty, it securely seeds dummy historical data.
2. **The Smart Risk Engine loop**: Vitals uploaded via `AddRecordForm` are evaluated in real-time. Manual wellness checks open localized support cards or populate the Intelligent Follow-Up queue.
3. **Alert Interception**: Any mathematical shift generated by the Risk Engine is caught by `AlertContext.jsx`, which routes to Toasts, Warnings, or Critical Popups.
4. **Emergency State & Recovery**: When a critical popup hits zero, it natively triggers `EmergencyModal.jsx`. Upon stabilizing, users click "Reset Risk" to clear the boolean cooldown loop and reset their baseline Risk Score to 10.

## 🛠 The Tech Stack
- **Frontend Core:** React.js 18 + Vite
- **Styling Architecture:** Tailwind CSS v4 using PostCSS integrations.
- **Routing:** React Router DOM v6.
- **Asset Library:** Lucide React for clean, dynamic SVGs and icons.
- **Persistence:** Local Storage Array Hydration `(Zero-Latency Cloud-less hackathon performance)`.
- **Backend Core:** Node.js + Express.
- **Generative Intelligence:** `@google/generative-ai` calling the `gemini-2.5-flash` model.

---

## 📁 Directory & Component Breakdown

- **`backend/server.js`**: Core AI routing engine exposing the `/api/analyze` Express endpoint.
- **`frontend/src/App.jsx`**: Wraps the Application Router and houses the `AlertProvider` context boundary.
- **`frontend/src/pages/Dashboard.jsx`**: The nervous system of the client application managing top-level states (`riskState`, `records`, etc.) and conditionally rendering all sub-pages.
- **`frontend/src/components/AlertContext.jsx`**: State machine holding `activeAlert`, queued notifications, and active `toasts`.
- **`frontend/src/utils/`**: Contains pure functional business logic like `RiskScoreManager.js`, `VitalMonitor.js` to handle the calculus of Risk Point modifiers, and `EmergencyManager.js`.
- **`frontend/src/components/`**: Houses all UI components (e.g. `SymptomChecker`, `WellnessPopup` for multistep wizard, `Dashboard UI Overlays` for popups and alerts, Secure Log Modules, and Data Charts).

## 🚀 Setup & Installation

**1. Clone the repository and install dependencies:**
```bash
# Frontend
cd frontend
npm install

# Backend
cd backend
npm install
```

**2. Configure Environment Variables:**
Inside the `backend/` folder, create a `.env` file and insert your active API Key:
```env
GEMINI_API_KEY=your_google_ai_studio_api_key_here
PORT=5000
```

**3. Booting Local Servers:**
You need to run both nodes concurrently.
```bash
# Terminal 1 - Frontend
cd frontend
npm run dev
# (Runs on typically port 5173)

# Terminal 2 - Backend
cd backend
node server.js
# (Runs on port 5000)
```

Have fun hacking! 🏆
